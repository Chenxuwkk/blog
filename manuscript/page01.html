# 分布式

# 分布式系统描述

  1. 多个处理单元互联的（核、cpu等都可以）
  2. 各种设备具有各种各样的连接方式
  3. 要做协同工作，并且完成一件事情，提供了某种服务

# 分布式4个指标

1. Scalability（balance）

  • 不要闲的闲死忙的忙死
  • 不要每台机器利用率小

2. Availability

  • 整体宕机了系统是否能继续提供服务
  • 发生宕机时，在用户的角度无感知

3. Consistency

  • 大家一致做同一个动作
  • 如何做到一致也是重点

4. Security

  • 数据完整性
  • 数据保密性
  • 认证和授权

# 进程间的通信

分布式计算的核心：（Inter-Process Communication，IPC）即在互相独立的进程间通信及共同协作以完成某项任务的能力。 在分布式计算环境中，两个或者多个进程按约定的某种协议进行IPC，特别的，一个进程可能即是发送方，也是接收方。 单播：一对一IPC 组播：一对另外一组（一对多）IPC

# 数据预处理

# 数据脏

  1. 不完整
    • 缺少数据值
  2. 有噪声
    • 包含错误或者孤立点
  3. 数据不一致
    • 命名或编码存在差异

# 预处理非常重要

# 数据质量度量

  1. 精确度
  2. 完整性
  3. 一致性
  4. 可信度
  5. 附加价值
  6. 可解释性

# 预处理任务

  1. 数据清理
    • 解决不一致性
  2. 数据集成
    • 多个数据库/集合成
  3. 数据变换
    • 规范化
  4. 数据归约
    • 得到的数据集压缩表示（找代表），比原始数据小的多，但结果相似

# 数据清理

  1. 填写空缺值
    • 人工填写：工作量巨大
    • 在所有的样本上，使用属性的平均值填充
    • 使用与给定元组属同一类的所有样本的平均值
  2. 识别离群点和平滑噪声
    • 计算机检测，人工判断
    • 回归：通过让数据适应回归函数来平滑
    • 聚类：监测并且去除孤立点
  3. 纠正不一致的数据
  4. 解决数据集成造成的冗余
