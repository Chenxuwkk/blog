# Git同步 😄

  • 创建远程仓库
  • 创建本地仓库
  • 本地仓库与仓库关联
  • 总结

本地仓上传

配置的是你个人的用户名称和电子邮件地址。这两条配置很重要，每次 Git 提交时都会引用这两条信息，说明是谁提交了更新,会随更新内容一起被永久纳入历史记录

git config --global user.name "Username" #替换Username
git config --global user.email  "xxx@.com" #替换邮箱

# 创建远程仓库

  1. 先在GitHub上创建一个远程仓库,仓库名为AutoTest

  2. 复制该仓库SSH地址

  3. git命令行模式 使用clone命令，克隆远程仓库到本地文件夹~/AutoTest

git clone git@github.com:Chenxuwkk/AotoTest.git

# 创建本地仓库

  1. 先在本地磁盘上新建一个文件夹(git_repository)并进入该文件夹，然后使用git init命令将该目录变成本地仓库
cd ~/
mkdir git_repository
cd ~/git_repository
git init
  1. 在Github上创建一个空的仓库

# 本地仓库与仓库关联

  1. 使用如下命令：
git remote add origin https://github.com/Chenxuwkk/AotoTest.git
  1. 把本地库的内容推送到远程仓库
git push -u origin master
  1. 若果本地仓库与远程仓库的内容不一致则会报错,则需要使用如下命令解决：
git pull --rebase origin master
git push origin master

该命令的意思是把远程库中的更新合并到（pull=fetch+merge）本地库中，--rebase的作用是取消掉本地库中刚刚的commit，并把他们接到更新后的版本库之中。

# 总结

本地文件目录为：

.
├── AutoTest
├── git_repository
